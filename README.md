### **1\. 概要**

#### **1.1. プロジェクト名**

PLATEAU CityGML \- Revitダイレクトインポーター

#### **1.2. 背景と目的**

建築・都市計画分野において、PLATEAUの3D都市モデル（CityGML）をRevitプロジェクトに取り込む需要が高まっています。
しかし、現状のワークフロー（Blender経由等）は複数のソフトウェアの専門知識を必要とし、非常に煩雑です。特に、生の測地座標データはRevitの作図限界（原点から約33km）を大きく超えるため、インポート時の座標処理が大きな障壁となっています。

本プロジェクトの目的は、これらの問題を**ワンクリックで解決**し、**複数回に分けたインポートでも位置関係が破綻しない**、Revit専用の**スタンドアロン・アドオン**を開発することです。

#### **1.3. 開発目標**

1. **時間短縮:** CityGMLのインポートにかかる作業時間を劇的に短縮する
2. **専門知識の不要化:** GISや座標系に関する専門知識が一切なくても、**安全かつ直感的**に都市モデルをインポートできるようにする
3. **コストゼロ:** サーバー費用やライセンス費用が一切かからない、**完全無料**のオープンソースツールとして提供する

### **3\. 機能要件 (アドオンが「何をするか」)**

| ID | 機能名 | 機能詳細 | 優先度 |
| :---- | :---- | :---- | :---- |
| FR-1 | **UIの追加** | Revitのリボンに新しいタブ「PLATEAU」と、その中に「CityGMLインポート」ボタンを追加する。 | **高** |
| FR-2 | **ファイル選択** | 「インポート」ボタンを押すと、ファイル選択ダイアログを表示する。 ・ 読み込み対象は .gml ファイル (CityGML 2.0形式) とする。 | **高** |
| FR-2.5 | **LOD選択** | ファイル選択後、詳細度レベル（LOD）を選択するダイアログを表示する（建物のみ）。 ・ **建物**: LOD1（簡易・高速）/ LOD2（詳細・推奨）を選択可能 ・ **道路・橋**: 利用可能な最大LODを自動選択（LOD2/LOD3） ・ データにLOD2が存在しない場合は自動的にLOD1にフォールバック | **高** |
| FR-3 | **GML解析** | 選択されたCityGML 2.0ファイル（XML）を解析し、建物・道路・橋の形状を構成する頂点座標データを抽出する。 ・ 座標系: EPSG:6697（JGD2011地理座標系）を自動検出し、ヒュベニの公式でメートル単位に変換 ・ **動的参照点方式（v1.1.0以降）**: GMLファイルの緯度経度バウンディングボックスの中心を参照点として自動計算 | **高** |
| FR-4 | **インテリジェント座標補正** | ユーザーが座標系を意識せず、Revitの作図限界（約33km）内に安全にインポートするための**自動座標補正**を行う。 **動的参照点方式（v1.1.0以降）:** ・ GMLファイルごとに最適な参照点（バウンディングボックスの中心）を自動計算 ・ 全国どこでも原点付近に正確に配置 ・ 初回インポート時の参照点をExtensible Storageに保存し、追加インポートで再利用 ・ Zオフセット永続化により、建物と道路の高さが自動的に一致 | **高** |
| FR-5 | **Revitジオメトリ生成** | 1\. 補正後の頂点座標データを基に、Revitプロジェクト内にDirectShapeとして3D形状を生成する。 2\. LOD優先順位制御: LOD2（詳細）> LOD1（簡易）、LOD0（2D足跡）は除外 3\. メッシュ生成: 多角形面を直接TessellatedFaceに渡す（三角形分割なし） 4\. 頂点溶接: 2.5mm (0.0082ft)、Target: Mesh、Fallback: Salvage | **高** |
| FR-6 | **ユーザーフィードバック** | 処理中の進捗を2段階で表示する。 ・ **Phase 1 (0-95%)**: CityGML解析と座標変換（随時更新） ・ **Phase 2 (95-100%)**: DirectShape生成（高速） ・ 完了時：インポート結果の詳細表示（建物数、エラー件数等） ・ 失敗時：エラーメッセージを表示 | **高** |

### **4\. 非機能要件 (アドオンの「品質」)**

| 項目 | 内容 |
| :---- | :---- |
| **ユーザビリティ** | \*\*「ボタンを押して、ファイルを選ぶ」以外の操作や設定を一切不要とする。マニュアルを読まなくても直感的に使えること。 |
| パフォーマンス | 一般的な50MB〜100MB程度のGMLファイル（例：新宿区LOD1）を、PCのスペックにもよるが、概ね1〜3分以内に処理完了できることを目標とする。 |
| 堅牢性 (ロバスト性) | ・ ユーザーがインポートしたモデルを手動で削除**した場合、そのモデルに紐づいていた「移動ベクトル」も**同時に消滅\*\*する。その後に別のモデルをインポートした場合、FR-4のロジックにより「初回インポート」として正しく処理されること。 ・ 原点から遠い（例: 八王子市）データでも、原点に近い（例: お台場）データでも、同じ操作で安全にインポートできること。 |
| **動作環境** | **Windows 10 / 11 Revit 2025以降** ・ Revit 2025以降を対象とし、.NET 8.0をターゲットフレームワークとする。 |
| **配布・保守性** | ・ 利用者がpyRevitや他のツールをインストールする必要がない、**完全スタンドアロン**のアドオンとする。 ・ 配布は、**インストーラー形式（MSI）** とする。 ・ 運用コスト（サーバー等）は一切発生しないこと。 |

### **5\. 開発スコープ（v1.0で やること・やらないこと）**

#### **5.1. スコープ内（v1.1.0で実装済み）**

* **建物 (bldg:Building)** のインポート (LOD1 および LOD2)
* **道路 (tran:Road)** のインポート (LOD1/LOD2/LOD3自動選択)
* **橋 (brid:Bridge)** のインポート (LOD2)
* **動的参照点方式による全国対応**（GMLファイルごとに最適な参照点を自動計算）
* **Zオフセット永続化**（建物と道路の高さを自動的に一致）
* **地物タイプ別マテリアル**（建物・道路・橋ごとに自動生成、色のカスタマイズ可能）
* 上記「3. 機能要件」で定義したすべての機能（特にインテリジェント座標補正）

#### **5.2. スコープ外（将来的な拡張候補）**

* **地形 (dem, tin)**: 地形データのインポートは将来バージョンでの対応を検討
* **テクスチャ:** 写真や模様などのテクスチャの再現は未対応（単色マテリアルのみ）
* **属性情報（メタデータ）:** 建物の名称、用途、高さなどの情報をRevitのパラメータとして付与する機能は、将来的な課題とする
* **LOD4の詳細モデル:** 室内構造を含むLOD4は現在のところ対応予定なし
* **モデルの編集機能:** 読み込んだモデルは背景用とし、Revit上で編集（壁や床への変換）はできない

- 道路の高さが0になる問題を修正
- 複数ファイルインポート時のZオフセット保存エラーを修正
