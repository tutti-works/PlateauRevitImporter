# PLATEAU CityGML to Revit インポーター 開発経緯

## プロジェクト概要
- **目的**: CityGML 2.0形式のPLATEAUデータをRevit 2025+にDirectShapeとしてインポート
- **環境**: Visual Studio 2022, Revit 2026, .NET 8.0
- **開発期間**: 2025年

## 主要な技術的課題と解決の試み

### 1. TessellatedShapeBuilder API互換性問題
**問題**: Revit 2025+でAPIが変更され、`Build()`が`void`を返すように変更
**解決**: `Build()`後に`GetBuildResult()`を呼び出す方式に変更

### 2. 進捗バーの実装（UI スレッド問題）
**問題**: WPF ProgressWindowの初期化タイミングでクラッシュ
**解決**: `ManualResetEvent`による同期化、`Dispatcher.BeginInvoke`の使用

### 3. DirectShape カテゴリエラー
**問題**: `Element id categoryId may not be used as a DirectShape category`
**解決**: 親カテゴリ（OST_GenericModel）を使用し、サブカテゴリは`Parameter`で設定

### 4. InternalException の連続発生（最大の問題）
**原因**:
- 頂点溶接の閾値が小さすぎた（0.3mm）
- Revitの内部公差（約2mm）より小さい距離の頂点が縮退ジオメトリを引き起こした

**解決**:
- 閾値を2.5mm (0.0082ft)に変更
- `TessellatedShapeBuilderTarget.Mesh` + `Fallback.Salvage`に変更

### 5. Extensible Storage の単位指定問題
**問題**: Revit 2025+で`Units are required for field`エラー
**試行1**: `SetSpec(SpecTypeId.Length)` → XYZ型と非互換
**試行2**: 3つのdoubleフィールドに分割 + `SetSpec(SpecTypeId.Number)` → 依然としてエラー
**現状**: Extensible Storageの保存を一時的に無効化

### 6. 座標軸の問題（解決済）✅
**現象**:
- 「縦に長い棒」のようなジオメトリが生成される
- デバッグログ: X/Y座標がほぼゼロ、Z座標だけが変化

```
座標範囲: X[-0.00, 0.00], Y[-0.01, -0.01], Z[7.48, 193.24]
座標範囲: X[0.00, 0.00], Y[-0.01, -0.01], Z[18.41, 58.87]
```

**根本原因** (Gemini Deep Research 2回目の調査で判明):
- `CalculateCentroid()`が全座標のX/Y平均値（重心）を計算
- オフセット = -重心 を適用すると、全建物が原点(0,0)周辺に集中
- **相対的な位置関係が失われる** → 建物のX/Y広がりが消失

**解決方法**:
- `CalculateCentroid()` → `CalculateBoundingBoxMin()` に変更
- X/Y/Z すべて**最小値**を使用（南西の角 + 地面レベル）
- これにより建物間の相対的な位置関係が保持される

**デバッグログ追加**:
- CityGML生座標の最初の3点をログ出力
- バウンディングボックス最小点をログ出力
- 計算されたオフセット値をログ出力

## 次のステップ（提案）

### 短期的対応（最優先）
1. **実際のCityGMLファイルでテスト**
   - デバッグログを確認（生座標、バウンディングボックス最小点、オフセット値）
   - 建物が正しい3D形状で表示されるか検証
   - 座標範囲がX/Y/Zすべてで正常な値を示すか確認

2. **ジオメトリ検証の改善**
   - 成功率を確認（以前：2-44個成功 / 13-3323個失敗）
   - 座標変換修正後の改善を測定
   - 必要に応じてさらなる調整

### 中期的対応
3. **Extensible Storageの再実装**
   - 文字列フィールドとしてJSON形式で保存
   - または、プロジェクトパラメータを使用

4. **パフォーマンス最適化**
   - 大容量ファイルの処理速度改善
   - メモリ使用量の最適化
