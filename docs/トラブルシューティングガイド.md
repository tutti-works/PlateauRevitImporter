## インストール時の問題

### ❌ 問題: 「別のバージョンが既にインストールされています」

**原因**: 古いバージョンのアドインが残っている

**解決方法**:

1. **既存バージョンをアンインストール**
   - Windows 設定 → アプリ → 「PLATEAU CityGML Importer for Revit」を検索
   - アンインストールをクリック

2. **手動でファイルを削除**（アンインストールが失敗する場合）
   ```
   C:\Program Files (x86)\PlateauRevitImporter\
   %ProgramData%\Autodesk\Revit\Addins\2025\PlateauRevitImporter.addin
   %ProgramData%\Autodesk\Revit\Addins\2026\PlateauRevitImporter.addin
   ```

3. **再インストール**

---

### ❌ 問題: 「管理者権限が必要です」

**原因**: インストーラーの実行に管理者権限が不足

**解決方法**:

1. `PlateauRevitImporter.msi` を右クリック
2. **「管理者として実行」** を選択

---

### ❌ 問題: インストールが途中で止まる

**原因**: ウイルス対策ソフトがブロックしている可能性

**解決方法**:

1. ウイルス対策ソフトを一時的に無効化
2. インストールを再試行
3. インストール後、ウイルス対策ソフトを再度有効化
4. 例外リストに以下を追加:
   ```
   C:\Program Files (x86)\PlateauRevitImporter\
   ```

## 起動・表示の問題

### ❌ 問題: Revitに「PLATEAU」タブが表示されない

**原因1**: Revitが起動中にインストールした

**解決方法**:
1. Revit を**完全に終了**（タスクマネージャーでプロセスを確認）
2. Revit を再起動

---

**原因2**: .addinファイルが正しく配置されていない

**解決方法**:

1. 以下のフォルダに `PlateauRevitImporter.addin` があるか確認:
   ```
   C:\ProgramData\Autodesk\Revit\Addins\2025\
   C:\ProgramData\Autodesk\Revit\Addins\2026\
   ```

2. ファイルが存在しない場合、手動でコピー:
   - ソース: `C:\Program Files (x86)\PlateauRevitImporter\PlateauRevitImporter.addin`（ただし、インストーラーディレクトリからコピー）
   - 宛先: 上記のフォルダ

---

**原因3**: DLLファイルが見つからない

**解決方法**:

1. 以下のファイルが存在するか確認:
   ```
   C:\Program Files (x86)\PlateauRevitImporter\PlateauRevitImporter.dll
   ```

2. 存在しない場合は、アドインを再インストール

---

### ❌ 問題: 「CityGML インポート」ボタンをクリックしても反応しない

**原因**: アドインの初期化エラー

**解決方法**:

1. Revit を再起動
2. 「ファイル」 → 「オプション」 → 「一般」 → **「起動時にアドインをロード」** にチェックが入っているか確認
3. Revit ジャーナルファイルでエラーを確認:
   ```
   %LOCALAPPDATA%\Autodesk\Revit\<バージョン>\Journals\
   ```

## インポート時の問題

### ❌ 問題: 「ファイルを読み込めませんでした」

**原因1**: ファイル形式が正しくない

**解決方法**:
- ファイル拡張子が `.gml` であることを確認
- CityGML 2.0 形式であることを確認（CityJSON や IFC は非対応）

---

**原因2**: ファイルが破損している

**解決方法**:
- PLATEAU VIEW または G空間情報センターから再ダウンロード
- ZIPファイルを解凍し直す

---

**原因3**: ファイルパスに日本語が含まれている

**解決方法**:
- ファイルを英数字のみのパス（例: `C:\Temp\`）に移動
- 再度インポートを試行

---

### ❌ 問題: 「XML形式が不正です」

**原因**: CityGML 1.0 や CityJSON など、非対応形式のファイル

**解決方法**:
- PLATEAU VIEW でダウンロード時に **「CityGML 2.0」** を選択
- CityJSON形式は未対応（将来のバージョンで対応予定）

---

### ❌ 問題: 「建物データが見つかりません」または「道路データが見つかりません」

**原因**: 選択した地物タイプとファイルの内容が一致していない

**解決方法**:
- ファイル名と選択した地物タイプが一致しているか確認
- 例:
  - 建物を選択 → `53393652_bldg_6697_op.gml` を選択（OK）
  - 道路を選択 → `53393652_tran_6697_op.gml` を選択（OK）
  - 橋を選択 → `53393652_brid_6697_op.gml` を選択（OK）
  - 建物を選択 → `53393652_tran_6697_op.gml` を選択（NG）

---

### ❌ 問題: 進捗バーが0%で止まる

**原因**: 非常に大容量のファイル（200MB以上）で、初期解析に時間がかかっている

**解決方法**:
- そのまま待機（5～10分程度かかる場合があります）
- タスクマネージャーで Revit のCPU使用率が0%でない場合、処理中です

---

### ❌ 問題: 「メモリ不足」エラー

**原因**: 非常に大容量のファイルでメモリが不足

**解決方法**:

1. **ファイルを分割**
   - 地区ごとに分けてインポート
   - 必要な範囲のメッシュのみダウンロード

2. **Revitを再起動**
   - メモリをクリアしてから再試行

3. **PCスペックの確認**
   - 推奨: RAM 16GB以上
   - 最低: RAM 8GB

## パフォーマンスの問題

### ⚠️ 問題: インポートが非常に遅い（10分以上）

**原因1**: ファイルサイズが非常に大きい（300MB以上）

**解決方法**:
- LOD1で試してみる（LOD2より高速）
- ファイルを地区ごとに分割

---

**原因2**: PCスペックが不足

**解決方法**:
- 他のアプリケーションを閉じる
- Revit以外のプロセスを終了
- PCを再起動してメモリを解放

---

**原因3**: デバッグモードでビルドされている（開発者向け）

**解決方法**:
- Release ビルドを使用しているか確認
- Debug ビルドは大量のログ出力により遅くなります

---

### ⚠️ 問題: Revitの動作が重くなった

**原因**: 大量のDirectShapeがインポートされている

**解決方法**:

1. **不要なモデルを削除**
   - 3Dビューで選択 → Delete

2. **ビューフィルターで非表示**
   - 表示/グラフィックス → フィルター → 「PLATEAU_Building」「PLATEAU_Road」「PLATEAU_Bridge」のチェックを外す

3. **ワークセットで管理**（大規模プロジェクトの場合）
   - ワークセットを作成し、PLATEAU モデルを別ワークセットに移動

---

## ジオメトリの問題

### ⚠️ 問題: モデルが表示されない、または一部しか表示されない

**原因1**: ビューの範囲外に配置されている

**解決方法**:
1. 3Dビューで **「ズームフィット」**（ZF）を実行
2. 「すべて表示」（ZA）を実行

---

**原因2**: ビューフィルターで非表示になっている

**解決方法**:
1. 「表示/グラフィックス」を開く
2. 「フィルター」タブで **「PLATEAU_Building」「PLATEAU_Road」「PLATEAU_Bridge」** にチェックが入っているか確認

---

**原因3**: GMLファイルに不正なジオメトリが含まれている

**解決方法**:
- 完了メッセージの **「エラー件数」** を確認
- エラー件数が多い場合、データ提供元に確認

---

### ⚠️ 問題: モデルの形状がおかしい（欠けている、歪んでいる）

**原因**: GMLデータの品質問題（PLATEAU側のデータ品質）

**解決方法**:
- **建物の場合**: LOD1とLOD2を切り替えて比較。LOD2で問題がある場合、LOD1を使用
- **道路の場合**: LOD3で立体形状、LOD1/LOD2は平面形状
- データ提供元（PLATEAU）に報告

---

### ⚠️ 問題: 建物と道路の高さが合わない

**原因**: 異なるZオフセットでインポートされている（旧バージョンの問題）

**解決方法**:
1. すべてのPLATEAUモデルを削除
2. v1.1.0以降を使用して再インポート
3. 建物を先にインポート → 道路を追加インポート（Zオフセットが自動的に再利用されます）

---

### ⚠️ 問題: モデルが重複している

**原因**: 同じファイルを2回インポートした

**解決方法**:
1. 重複したモデルを手動で削除
2. 削除方法:
   - 3Dビューで選択 → Delete
   - または「選択」 → 「カテゴリ別」 → 「PLATEAU_Building」「PLATEAU_Road」「PLATEAU_Bridge」 → Delete

---

## 座標・位置の問題

### ⚠️ 問題: 建物が原点から遠く離れた位置に配置される（v1.0.xの問題）

**原因**: v1.0.x以前のバージョンでは固定参照点方式を使用していたため、お台場以外の地域で座標ドリフトが発生していました

**解決方法**:
1. **v1.1.0以降にアップグレード**
   - 動的参照点方式により、GMLファイルごとに最適な参照点を自動計算
   - 全国どこでも原点付近に正確に配置されます
2. すべてのPLATEAUモデルを削除
3. v1.1.0で再インポート

---

### ⚠️ 問題: 2回目のインポートで位置がズレた

**原因**: 通常はズレません。以下を確認:

**確認方法**:
1. 既存のモデルを削除していないか確認
2. 異なる座標系のファイルを混在させていないか確認（すべてEPSG:6697であるべき）

**解決方法**:
- すべてのモデルを削除し、最初からインポートし直す

---

### ✅ 正常動作: モデルが原点付近に配置される（v1.1.0以降）

**仕組み**:
- v1.1.0以降は動的参照点方式を採用
- GMLファイルの緯度経度のバウンディングボックス中心を参照点として自動計算
- 参照点からの相対座標で配置されるため、常に原点付近に配置されます
- 複数地区をインポートしても、初回インポート時の参照点が再利用され、相対位置が保たれます

## エラーメッセージ一覧

### 🔴 「CityGMLファイルの解析中にエラーが発生しました」

**原因**: XML解析エラー、ファイル破損

**解決方法**:
- ファイルを再ダウンロード
- テキストエディタでXML構文エラーを確認

---

### 🔴 「座標変換中にエラーが発生しました」

**原因**: 座標データが不正、または範囲外

**解決方法**:
- ファイルの座標系が EPSG:6697 であることを確認
- 日本国外のデータは未対応

---

### 🔴 「ジオメトリ生成中にエラーが発生しました」

**原因**: Revit APIのジオメトリ生成エラー

**解決方法**:
- Revit を再起動
- LOD1とLOD2を切り替えて試す
- 問題が継続する場合、データ品質の問題

---

### 🔴 「トランザクションのコミットに失敗しました」

**原因**: Revitのトランザクション競合

**解決方法**:
- インポート中に他の操作をしない
- Revit を再起動

---

## ログとデバッグ情報

### Revit ジャーナルファイル

エラーの詳細情報は Revit ジャーナルファイルに記録されます：

**場所**:
```
C:\Users\<ユーザー名>\AppData\Local\Autodesk\Revit\Autodesk Revit <バージョン>\Journals\
```

**確認方法**:
1. 最新の `.txt` ファイルを開く
2. `PLATEAU` または `PlateauRevitImporter` で検索
3. エラーメッセージやスタックトレースを確認

---

### デバッグビルド（開発者向け）

開発環境でデバッグビルドを使用すると、詳細なログが出力されます：

1. Visual Studio で **Debug モード** でビルド
2. **Output ウィンドウ** で `Debug` を選択
3. CityGML解析、座標変換の詳細ログを確認

例:
```
CityGML生座標（緯度・経度） #1: 緯度=35.6895000, 経度=139.6917000, 高さ=10.50m
変換後相対座標（メートル） #1: X=6789.12m, Y=-10234.56m, Z=10.50m
```
